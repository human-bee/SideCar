# Design Kit

SideCar should feel native to Codex Desktop while reducing visual noise.

## Principles

- Status first, then summary, then evidence.
- The timeline behaves like a semantic map: summaries at low zoom, tool/command detail as the user zooms in.
- Thread mutation is never hidden; every action appears as a target card before execution.
- Light and dark appearance should mirror the current Codex app appearance.
- Voice is quiet by default and becomes speech-to-speech only when toggled.

## MVP Surfaces

- Floating side panel.
- Header with active thread, state, and freshness source.
- Header icon navigation for Active, Threads, Talk, and Settings.
- Status summary.
- Zoomable timeline map.
- Recommendations and next action.
- Talk surface with target thread identity, queue, steer, `/side`, Realtime readiness, and screen-preview controls.
- Confirmation cards for staged actions.
