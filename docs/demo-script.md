# Two Minute Demo Script

## Setup

- Codex Desktop is visible with this SideCar build thread selected.
- SideCar is open beside it.
- Press Option-Shift-Space to jump to Talk mode and check Realtime readiness.

## Talk Track

**0:00 - 0:20**

"SideCar is a local companion for a running Codex thread. The point is not to replace the main agent. The point is to give me a second, lightweight assistant while Codex is busy."

**0:20 - 0:45**

"This thread is the build process for SideCar itself. The panel tells me which Codex run I am attached to, whether the source is live or needs refresh, and what the current run is doing."

**0:45 - 1:10**

"The chat box is the main surface. If I ask a tangent, SideCar can answer in its own context. If I want to affect the main Codex run, I stage a queue, steer, or side action and confirm it."

**1:10 - 1:35**

"The important safety boundary is that SideCar does not silently mutate the running thread. Actions become target cards first: thread, action, payload, and confirmation."

**1:35 - 2:00**

"For the demo, the realtime voice layer is the future interaction model. I can speak to SideCar about this exact Codex run, summarize how we built it, and decide whether to queue, steer, fork, or leave the main agent alone."

## Demo Questions

- "What did this Codex thread build so far?"
- "What is the safest next step before sharing this with OpenAI?"
- "Stage a side question asking what the current architecture risk is."
- "Summarize the difference between queue, steer, and side."
